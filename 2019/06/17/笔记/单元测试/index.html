单元测试

  • jest安装
    • npm install –save-dev jest

  • 使用匹配器

    • 最简单的测试值的方法就是看是否精准匹配,首先是toBe()

  • toBe用的是JavaScript中的Obiect.is(),属于Es6中的特性,所以不能检测对象,如果需要检测对象的值的话,需要用到toEqual,toEqual递归检查对象或者数组中的每个字段.

  • 在实际测试中,我们有时候需要区分undefined,null,false,以下的一些规则有助于我们进行

    1. toBeNull只匹配null
    2. toBeUndefined只匹配undefined
    3. toBeDefine与toBeUndefined 相反
    4. toBeTruthy匹配任何if语句为真
    5. toBeFalsy匹配任何if语句为假
  • 大多数的比较数字有等价的匹配器
    1. 大于toBeGreaterThan()
    2. 大于或者等于toBeGreaterThanOrEqual()
    3. 小于 toBeLessThan()
    4. 小于或者等于 toBeLessThanOrEqual()
      5.toBe和toEqual同样适用于数字
      注意: 对比两个浮点数是否相等的时候,使用toBeCloseTo而不是toEqual

  • 字符串

    • 使用toMatch()测试字符串,传递的参数是正则表达式

  • toContain()可以检测数组中是否包含某一特定项

  • 另外 如果你想在测试特定函数的时候抛出一个错误,在它调用的时候可以使用toThrow

    测试异步代码

  • 注意:回调函数和异步没有什么必然联系,回调只是异步的一种调用方式而已,不要异步和回调两个概念结合起来.

  • 默认情况下一旦到达运行上下文的底部,jest测试就会立即结束,这意味着这个测试将不能按照预期的进行,使用单个的参数调用done,而不是将测试放在一个空参数的函数中,jest会等done回调函数执行结束后,结束测试.

  • 如果done()永远不会被调用,则说明这个测试将失败,这也正是我们所希望看到的.

    peomise

  • 如果代码中使用到了promise,只需要从你的测试中返回一个promise,jest就会等待这个promise来解决,如果承诺被拒绝,则测试将会自动失败

  • 举例 如果fetchData,使用Promise代替回调的话,返回值是应该解析为一个字符串’peanut butter’的Promise,那么可以用以下方式测试代码

  • 另一种情况就是,想要Promise被拒绝,我们可以使用.catch方法.另外，要确保添加了expect.assertions来验证一定数量的断言被调用。否则一个fulfilled态的Promise不会让测试失败。

.resolves/.rejects

  • 可以使用.resolves匹配器匹配你的期望的声明,如果想要被拒绝,可以使用rejects

    jest.mock()

  • mock函数提供的以下三种特性
    1. 捕获函数调用情况
    2. 设置函数返回值
    3. 改变函数的内部实现
  • jest.fn()
    1.jest.fn()是创建Mock函数最简单的方式,如果没有定义函数内部的实现,jest.fn()会返回undefined作为返回值
