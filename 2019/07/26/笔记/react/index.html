react

  • React 是一个用于构建用户界面的声明性,高效且灵活的 javascript 库,使用 React 可以将一些简短、独立的代码片段组合成复杂的 UI 界面，这些代码片段被称作“组件”
  • React 中有多种不同类型的组件
  • 其中 ShoppingList 是一个 React 组件类,或者说是一个 React 组件类型,一个组件接收一些参数,我们把这些参数叫做 props,然后通过 render 方法返回需要展示在屏幕上的视图层次结构.
  • render 方法返回值描述了你希望在屏幕上看到的内容,react 根据描述,然后把结果展示出来,更具体的说,render 返回了一个 react 元素,这是一种对渲染内容轻量级的描述,大多数的 React 开发者使用一种名为”JSX”的特殊语法,JSX 可以让你更轻松的书写这些结构,语法

  • 在 JSX 中你可以任意使用 JAVASCRIPT 表达式,只需要用一个大括号把表达式括起来每一个 React 元素事实上都是一个 JavaScript 对象,你可以在你的程序中把它当保存在变量中,或者当作参数传递
  • 在 javascript class 中,每次你定义子类组件时都需要调用 super 方法,因此,在所有含有构造函数的 React 组件中,构造函数必须以 super(props)开头
  • 当你遇到需要同时获得多个子组件数据时,或者两个组件之间需要相互通讯的情况时,需要把子组件的 statr 数据提升至其共同的父组件当中保存,之后父组件可以通过 props 将状态数据传递到子组件当中,这样应用当中所有组件的状态数据就能更方便的同步共享了

JSX 简介

  • const element = <h1>Hello, world!</h1>;
    • 这个标签语法是 JavaScript 的语法拓展,JSX 可以很好的描述 UI 应该呈现出它应有交互的本质形式,JSX 可能会使人联想到模板语言,但他具有 javascript 的全部功能
  • React 认为渲染逻辑本质上与其他 UI 逻辑内在耦合,比如,在 UI 中需要绑定处理事件,在某些时刻状态发生变化时需要通知到 UI,以及需要在 UI 中展示准备好的数据
  • 在 JSX 中嵌入表达式
  • JSX 也是一个表达式,在编译之后,JSX 表达式会被转化为普通的 JavaScript 函数调用,并且对其取值后得到 JavaScript 对象,也就是说,你可以在 if 语句和 for 循环的代码块中使用 JSX,将 JSX 赋值给变量,把 JSX 当作参数传入,以及从函数中返回 JSX
  • JSX 特定属性 1.你可以通过使用引号,来将属性值指定为字符串字面量:const element = <div tabIndex="0"></div>;
    1. 也可以使用大括号,来在属性值中插入一个 JavaScript 表达式:const element = <img src={user.avatarUrl}></img>;
    2. 在属性中嵌入 JavaScript 表达式时,不要在大括号外面加上引号,应该仅使用大括号或者引号其中的一个,对于同一属性不能同时使用这两种符号
  • 因为 JSX 语法上更接近 JavaScript 而不是 HTML,所以 React DOM 使用 camelCase(小驼峰命名)来定义属性的名称,而不使用 HTML 属性名称的命名约定
  • 使用 JSX 指定子元素
    1. 假如一个标签里面没有内容,你可以使用/>来闭合标签,就像 XML 语法一样
      const element = <img src={user.avatarUrl} />;
    2. JSX 标签里能够包含很多子元素
