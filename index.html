bom

  • offset家族 offsetWidth offsetHeight offsetTop offsetLeft offsetParent
  • offsetWidth offsetHeight盒子自身的宽度和高度加上自身的padding和border
  • offsetTop offsetLeft检测盒子距离父类定位盒子的距离，如果父类没有定位则一直找到body
  • offsetLeft和.style.left的区别
    • 前者返回的是num类型可以直接参与运算，但是是只读类型的，不能通过它写入。
    • 后者返回的是字符串类型的，不能直接参与运算，但是可以通过它写入
  • offsetParent 检测父系盒子中具有定位元素的盒子节点，返回标签名称

    匀速运动

  • 匀速运动的过程
    • 求当前位置
    • 计算运动后的当前位置
    • 赋值
    • 计算差值
    • 停止条件
    • 如果差值小于步长则直接跳到目标位置

缓动

  • 缓动公式 leader = leader + (target - leader)/10;
  • 缓动过程为
    • 求当前位置
      • 求步长
        • 判断如果步长小于1则步长为1，
      • 求运动后的当前位置
      • 赋值
      • 停止条件

  • scrollTop 和 scrollLeft 指的是被body卷进去的高度和宽度

      client 对象

  • clientWidth clientHeight获取网页可视区域的宽度和高度，如果盒子调用的话指的是盒子本身的宽度和高度
  • clientX 鼠标距离可视区域左侧距离
  • clientY 鼠标距离可视区域上侧的距离
  • clientTop/clientLeft 盒子的border宽高

    三大家族的区别

  • width 和 height
    • clientWidth = width + padding
    • clientHeight = height + padding
    • offsetWidth = width + padding + border
    • scrollWidth = 内容的宽度 （不包含border）(能被卷进去的东西，如果超出计算整个内容)
  • top 和left
    • offsetTop 调用者（任意元素，盒子为主）距离父系盒子中带有定位元素的距离
    • scrollTop : 调用者 document.documentElement 作用：浏览器无法显示被卷进去的部分
    • clientTop 调用者：盒子
      作用：指的是border
    • clientX 调用者：event
      作用：指的是鼠标距离浏览器可视区域的距离

      client 家族特殊用法，检测浏览器的宽/高

  2. window.innerHeight

浮动

  • 块级元素和行内元素都可以浮动，当一个行内元素浮动以后将会自动变为一个块级元素，当一个块级元素浮动以后，宽度会默认被内容撑开，所以当漂浮一个块级元素时我们都会为其指定一个宽度。 当一个元素浮动以后，其下方的元素会上移。元素中的内容将会围绕在元素的周围，元素浮动以后即完全脱离文档流，这时不会再影响父元素的高度。也就是浮动元素不会撑开父元素。浮动元素默认会变为块元素，即使设置display:inline以后其依然是个块元素。
    如果子类元素设置了浮动，而父类元素没有设置高度，或者高度比子类元素小，那么子类元素脱离了文档流，就无法把父类盒子撑开。那么整个文档的结构将受到破快。清除浮动的影响： clear: left/right/both 不允许当前元素的left/right/both有浮动元素。在浮动元素的最后面追加一个空的div,设置clear:both即可清除浮动的影响。
  • 一般只是用左浮动，因为float-right会改变标签的前后顺序。
  • 一旦子类发生了浮动 父类盒子不需要写高度
