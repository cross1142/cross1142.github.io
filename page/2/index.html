bom

  • offset家族 offsetWidth offsetHeight offsetTop offsetLeft offsetParent
  • offsetWidth offsetHeight盒子自身的宽度和高度加上自身的padding和border
  • offsetTop offsetLeft检测盒子距离父类定位盒子的距离，如果父类没有定位则一直找到body
  • offsetLeft和.style.left的区别
    • 前者返回的是num类型可以直接参与运算，但是是只读类型的，不能通过它写入。
    • 后者返回的是字符串类型的，不能直接参与运算，但是可以通过它写入
  • offsetParent 检测父系盒子中具有定位元素的盒子节点，返回标签名称

    匀速运动

  • 匀速运动的过程
    • 求当前位置
    • 计算运动后的当前位置
    • 赋值
    • 计算差值
    • 停止条件
    • 如果差值小于步长则直接跳到目标位置

缓动

  • 缓动公式 leader = leader + (target - leader)/10;
  • 缓动过程为
    • 求当前位置
      • 求步长
        • 判断如果步长小于1则步长为1，
      • 求运动后的当前位置
      • 赋值
      • 停止条件

        scroll 家族

  • scrollWidth指的是盒子的宽度只包含padding和内容的宽度
  • scrollTop 和 scrollLeft 指的是被body卷进去的高度和宽度
  • scrollTop 可以设置值

    骨架标签

  • document.title — 文档标题；
    document.head — 文档的头标签
    document.body — 文档的body标签；
    document.documentElement — html
  • onscroll 可以监听浏览器的滚动事件和onclick的用法相同

    event对象

  • 时间对象的获取
    • 在ie6,7,8中事件对象的获取用window.event,在高级浏览器中在绑定事件的函数中加参数，这个形参就是event.
  • event 属性
    • pageX/pageY指的是相对于页面本身的位置坐标
    • screenX/screenY指的是相对于显示器的位置坐标
    • clientX/clientY指的是浏览器客户端（不包含top的工具栏）

      client 对象

  • clientWidth clientHeight获取网页可视区域的宽度和高度，如果盒子调用的话指的是盒子本身的宽度和高度
  • clientX 鼠标距离可视区域左侧距离
  • clientY 鼠标距离可视区域上侧的距离
  • clientTop/clientLeft 盒子的border宽高

    三大家族的区别

  • width 和 height
    • clientWidth = width + padding
    • clientHeight = height + padding
    • offsetWidth = width + padding + border
    • scrollWidth = 内容的宽度 （不包含border）(能被卷进去的东西，如果超出计算整个内容)
  • top 和left
    • offsetTop 调用者（任意元素，盒子为主）距离父系盒子中带有定位元素的距离
    • scrollTop : 调用者 document.documentElement 作用：浏览器无法显示被卷进去的部分
    • clientTop 调用者：盒子
      作用：指的是border
    • clientX 调用者：event
      作用：指的是鼠标距离浏览器可视区域的距离

      client 家族特殊用法，检测浏览器的宽/高

  1. docuemnt.body.clientHeight
  2. window.innerHeight

    如果子类元素设置了浮动，而父类元素没有设置高度，或者高度比子类元素小，那么子类元素脱离了文档流，就无法把父类盒子撑开。那么整个文档的结构将受到破快。清除浮动的影响： clear: left/right/both 不允许当前元素的left/right/both有浮动元素。在浮动元素的最后面追加一个空的div,设置clear:both即可清除浮动的影响。

简介

    set结构

  • Es6提供了新的数据结构set,它类似于数组但是成员的值都是唯一的,没有重复的值,(不包括空对象),Set本身是一个构造函数,用来生成Set数据结构

  • 解构赋值允许指定默认值

  • Set函数可以接受一个数组作为参数,用来初始化

  • 在set内部任为两个NAN是相等的
  • 两个对象总是不相等的,所有set里两个空对象被视为两个值
  • Set 结构的实例有以下属性。
    • Set.prototype.constructor：构造函数，默认就是Set函数。
    • Set.prototype.size：返回Set实例的成员总数。
  • set实例方法
    • add(value)：添加某个值，返回 Set 结构本身。
    • delete(value)：删除某个值，返回一个布尔值，表示删除是否成功。
    • has(value)：返回一个布尔值，表示该值是否为Set的成员。
    • clear()：清除所有成员，没有返回值。
  • set的遍历方法
    • keys()返回键名的遍历器
    • values()返回键值的遍历器
    • entries()返回键值对的遍历器
    • forEach()使用回调函数遍历每个成员
  • 需要特别指出的是，Set的遍历顺序就是插入顺序。这个特性有时非常有用，比如使用 Set 保存一个回调函数列表，调用时就能保证按照添加顺序调用。
  • 可以省略values方法，直接用for…of循环遍历 Set。
  • Array.from方法可以将 Set 结构转为数组

  • 扩展运算符和 Set 结构相结合，就可以去除数组的重复成员。

  • 数组的map和filter方法也可以间接用于 Set 了

  • 使用 Set 可以很容易地实现并集（Union）、交集（Intersect）和差集（Difference）。

  • 在遍历操作中，同步改变原来的 Set 结构

  • 在ES6中，会默认采用严格模式，因此this也不会自动指向window对象了，而箭头函数本身并没有this，因此this就只能是undefined，这一点，在使用的时候，一定要慎重慎重再慎重，不然踩了坑你都不知道自己错在哪！这种情况，如果你还想用this，就不要用使用箭头函数的写法。
